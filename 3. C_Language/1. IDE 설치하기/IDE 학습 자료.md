## 1. IDE가 뭘까?
: IDE란 '통합 개발 환경'으로, 개발자가 프로그램을 작성하고 실행하기 쉽게 여러 도움을 주는 도구를 말합니다.

우리가 Python 수업에서 써 보았던 파이참, 코랩의 .ipynb, IDLE과 같은 것들을 IDE라고 합니다.

앞으로 'IDE'라는 표현을 정말 많이 사용할 건데, IDE라는 표현이 나왔을 때, 파이참 / 코랩 / IDLE과 같은 코딩 도구라고 생각하면서 글을 읽어 나가면 됩니다.

오늘 수업에서는, IDE를 왜 사용해야 하는지에 대해 한 번 더 생각해 보고, C언어를 개발하기 위한 'Eclipse'라는 IDE를 설치해 볼 겁니다.

## 2. IDE는 언제, 왜 쓰는 걸까?
: 우선, IDE를 왜 사용하는지에 대해 생각해 볼 필요가 있습니다. 여러분들은 왜 Python에서 파이참, 코랩, IDLE과 같은 도구들을 사용했나요?

그렇게 하지 않으면 파이썬을 실행할 수 없다고 생각했을 수도 있습니다. 하지만, 메모장 하나만으로도 python 파일을 실행시킬 수 있습니다.

그렇다면 왜 우리는 메모장으로 코딩하지 않았을까요?

우리가 파이참, 코랩, IDLE과 같은 환경에서 코딩했던 이유는, 개발을 더 쉽게 하기 위함이었습니다.

코드를 짜다가 오타를 냈다면, 오타가 났을 때 빨간 줄로 경고를 줘서 어디가 틀렸는지에 대한 힌트를 주기도 합니다. 하지만 메모장은 그런 힌트를 일절 제공하지 않습니다.

오늘 수업의 마지막 즈음에서는, 메모장으로 코딩을 해볼 겁니다. 메모장으로 파이썬 코드나 C언어 코드를 작성하고, 터미널을 통해 실행해 보면서, 왜 IDE를 써야 하는지 피부로 느껴 보는 시간을 가질 겁니다.

근데 IDE에는 오타를 잡는 기능만 있는 걸까요? IDE가 제공하는 여러 기능에 대해 조금 더 알아보도록 하겠습니다.

## 3. IDE는 어떻게 쓰는 걸까?
: IDE가 제공하는 기능은 엄청 많습니다. 코드에서 오타를 알려줄 뿐만 아니라, 실행하면서 변수에 값이 얼마로 저장되어 있는지에 대한 기록도 남깁니다. ('디버깅'이라고 표현합니다.)

테스트, 배포와 같은 기능들도 지원하지만 당분간 우리는 사용하지는 않을 계획입니다. 언어에 충분히 능숙해지고 난 뒤에, 프로젝트를 경험해 보면서 사용하는 것으로 하겠습니다.

일단은 IDE를 먼저 설치하고, 나중에 기회가 되면 백준 문제 풀이와 함께 '디버깅'이라는 기능에 대해서도 다뤄 보겠습니다.

"개발하는 데에 많은 도움을 준다"는 것만 알아 두고, 일단 바로 설치로 넘어가도록 하겠습니다.

## 6. [심화 : 작성, 컴파일, 실행은 다 다른 말이다.]
### 6-1. 코드, 컴파일 그리고 기계어

<span style="color:pink">**😉 너무 어려우면 넘어가도 괜찮아요 😉**</span>

Python을 경험해 봤다면, '실행' 버튼을 많이 눌러 봤을 것 같습니다. 파이참에서는 초록색 삼각형 모양이 '실행' 버튼이고, 코랩에서는 엔터로 구분하고, IDLE에서는 위 목록 창에서 확인했을 겁니다.

우리가 터미널에서 gcc를 치고도, hello라고 한 번 더 쳤던 이유는, '실행'은 한 번의 단계로 질행되지 않기 때문입니다.

'컴파일'이라는 첫 번째 단계를 거치고, '실행'이라는 두 번째 단계를 꼭 거쳐야 합니다.

너무 어려운 절차라고 생각되면 넘어 가도 돼요. 하지만 두 번째로 교재를 보고 있는 거라면, 설명까지는 하지 못하더라도 이해는 해 봤으면 좋겠어요.

---

![기계어와 어셈블리어](/3.%20C_Language/1.%20IDE%20설치하기/기계어와%20어셈블리어.png)

우리는 일반적으로 '코드'를 씁니다. 이 코드는 '컴퓨터'에게 명령을 내립니다. 사진 상에서 '고급언어'라고 써있는 부분이, '코드'라고 할 수 있겠습니다.

우리가 앞서 사용한 gcc는 우리가 작성한 코드를 어셈블리어로 먼저 바꿉니다. 그 다음에 기계어로 바꿔주게 됩니다.

기계어는 0과 1로만 이루어진 숫자의 연속이고, 어셈블리어는 사람이 이해할 수 있는 약간의 영어가 섞인 명령문입니다.

가장 중요하게 알아야 할 것은, '컴파일'과 '실행'의 차이라고 할 수 있겠습니다. '컴파일'은 우리가 작성한 코드를 기계어 혹은 어셈블리어의 형태로 바꾼 것을 말합니다.

그리고 '실행'은 '기계어'를 실행하는 것을 의미합니다. 컴파일만 한다고 해서, 프로그래밍을 실행시킬 수 없습니다. 우리가 터미널에서 gcc라는 명령어를 쳤을 때, 실행이 되지 않는 것도 gcc는 그저 컴파일만 하기 때문입니다.

이 내용은 컴퓨터 공학과 고학년의 수업인 '컴퓨터 구조'에서 가장 기초적으로 다루는 이야기 중 하나입니다. 이해가 되지 않는다면, "컴파일과 실행은 다르다"는 것만 이해하고 넘어가도 좋습니다.

## [8. 심화 - C언어를 배우면]

<C언어의 장점>

[1. 다른 언어를 배우기가 쉬워진다]

: C언어를 기반으로 한 언어들이 많기 때문이에요. (자료형 등) 

[2. 컴퓨터의 메모리를 직접적으로 조작할 수 있다]

: 내가 필요한 만큼의 메모리를 사용하기 때문에, 컴퓨터를 효율적으로 사용할 수 있게 된다.

[3. Java, Python과 같은 언어들에 비해서 실행 속도가 훨~씬 빠르다]

: 프로그램을 만들기 위해 많은 것들이 필요하지 않아요.

---

<C언어의 단점>

[1. 배우기에 어려울 수 있다]

: 자료형을 모두 선언하고, 환경 설정을 해줘야 할 것들이 파이썬 같은 언어들에 비해 많아요.

[2. 직접 해줘야 할 것들이 많다]

: 파이썬에서는 컴퓨터가 알아서 해줬던 것들이 많아요. 하지만, C언어는 컴퓨터가 알아서 해주지 않는 경우가 굉~장히 많아요.

변수를 선언할 때도 자료형을 붙여야 하고, 함수 같은 것들을 호출할 때도 세미콜론 -> ; 을 꼭 붙여줘야 하는 경우가 많아요.

규칙이 많지만, 그만큼 정확하게 표현해야 하기 때문에, 오히려 편한 면도 많아요. 아직은 이해가 안 될지 몰라도, 앞으로 수업을 들어 가면서 느껴보도록 하면 좋을 것 같아요!

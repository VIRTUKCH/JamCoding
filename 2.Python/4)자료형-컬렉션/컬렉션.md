## 1. 컬렉션이 뭘까?
: 이번 시간에 배울 건 컬렉션입니다. 컬렉션이란, 값들을 모아서 저장할 수 있는 자료 구조를 말해요.

컬렉션은 자료형이라고 불리기도 하면서, 데이터 구조라고 불리기도 해요.

자주 사용하는 컬렉션으로는 리스트(list), 튜플(tuple), 집합(set), 딕셔너리(dictionary) 등이 있습니다.

이번 시간에는 자주 사용되는 컬렉션에 대해 알아 볼 겁니다.

특히, 다른 컬렉션보다 리스트에 조금 더 집중해서 알아 보도록 할게요.

#### [*참고]
우리가 그 전에 배웠던 정수형, 문자형과 같은 경우에는 자료형이라고 불리기는 하지만, 데이터 구조에는 해당하지 않아요.

아직은 감이 잘 오지 않을 수 있지만, 일단은 지금부터 배울 것들이 자료형임과 동시에 데이터 구조라는 것만 알아 두고 넘어 가도록 할게요.

* 자료 구조(데이터 구조, Data Structure) : 데이터의 모임, 또 데이터 간의 관계, 그리고 데이터에 적용할 수 있는 함수나 명령

---

#### 1. 리스트 : 여러 가지 자료들을 순서대로 모아둔 자료형이에요. 대괄호([]) 안에 자료들을 나열하면 돼요.
```python
list_a = [1, 2, 3, 4, 5] # 리스트는 괄호를 이용해서 선언합니다.
list_b = [6, 7, 8, 9, 10]
print(list_a) # 결과를 확인해 보자.
print(list_b) # 결과를 확인해 보자.
print(list_a + list_b) # 결과를 확인해 보자.
```

#### 2. 튜플 : 리스트와 비슷하지만, 한 번 만들면 변경할 수 없어요. 소괄호(())을 사용해요.
```python
tuple_a = (1, 2, 3, 4, 5)
tuble_b = (6, 7, 8, 9, 10)
print(tuple_a)
print(tuple_b)
print(tuple_a + tuple_b) # 결과를 확인해 보자.
```

#### 3. 딕셔너리 : '키'와 '값'의 쌍으로 이뤄진 자료형이에요. 중괄호({})를 사용하고, 키를 이용해 값을 찾을 수 있어요.
```python
dictionary_a = {"이름" : "유재석", "나이" : 5}
dictionary_b = {"apple" : "사과", "banana" : "바나나"}
print(dictionary_a)
print(dictionary_b)
print(dictionary_a + dictionary_b) # 결과를 확인해 보자.
```

## 2. 리스트는 언제, 왜 쓸까?

: 리스트는 파이썬에서 가장 자주 사용되는 컬렉션 중 하나입니다. 자주 사용되는 이유에는 다음과 같은 이유들이 있어요.

#### 1. 여러 데이터를 모아서 관리할 때 사용하기 좋다.
: 예를 들어, 친구들의 이름, 성적 목록, 할 일 목록 등을 한데 모아서 관리하고 싶을 때 리스트를 사용할 수 있습니다.
```python
friends = ['지민', "태형", "정국", 1, 2.5, 3.14, True, False]
print(friends) # 결과를 확인해 보자.
```

#### 2. 데이터의 순서가 중요할 때 사용하기 좋다.
: 리스트에 데이터를 추가하면, 그 순서가 그대로 유지됩니다. 따라서 순서대로 데이터를 관리하거나 접근해야 할 때 유용합니다.
```python
grades = [92, 85, 100, 78, 85]
print(grades) # 결과를 확인해 보자.
```

#### 3. 데이터를 자주 변경해야 할 때 사용하기 좋다.
: 리스트는 데이터를 추가, 삭제, 수정하기 편리합니다. 따라서 데이터가 자주 변경되는 상황에서 리스트를 사용하면 좋습니다.
```python
todo_list = ['숙제하기', '방청소', '책읽기']
todo_list.append('운동하기')  # 할 일 추가
print(todo_list) # 결과를 확인해 보자.
todo_list.remove('방청소')  # 할 일 삭제
print(todo_list) # 결과를 확인해 보자.
```

## 3. 리스트가 가지고 있는 함수들
: 아직 함수에 대해 배우지는 않았어요. 하지만, 리스트를 잘 활용하기 위해서는 리스트를 통해 사용할 수 있는 함수들에 어떤 것들이 있는지 아는 게 좋아요.

함수에 대해서는 조금 있다가 배울 계획이에요. 일단은 아래 예제들을 보면서, 이런 것들도 있구나~ 하는 마음으로 리스트의 함수들을 구경해 보면 좋겠어요.

---

#### 1. append(x): 리스트의 끝에 항목 x를 추가합니다.
```python
fruits = ['apple', 'banana']
fruits.append('cherry')
print(fruits)  # ['apple', 'banana', 'cherry']
```

#### 2. extend(iterable): 리스트의 끝에 모든 iterable의 항목을 추가합니다. iterable은 리스트, 튜플, 딕셔너리 등이 될 수 있습니다.
```python
fruits = ['apple', 'banana']
more_fruits = ['cherry', 'date']
fruits.extend(more_fruits)
print(fruits)  # ['apple', 'banana', 'cherry', 'date']
```

#### 3. insert(i, x): 지정된 위치 i에 항목 x를 삽입합니다.
```python
fruits = ['apple', 'banana', 'cherry']
fruits.insert(1, 'blueberry')
print(fruits)  # ['apple', 'blueberry', 'banana', 'cherry']
```

#### 4. remove(x): 리스트에서 첫 번째로 나오는 x를 삭제합니다. x가 리스트에 없으면 오류가 발생합니다.
```python
fruits = ['apple', 'banana', 'cherry']
fruits.remove('banana')
print(fruits)  # ['apple', 'cherry']
```

#### 5. pop([i]): 지정된 위치 i의 항목을 리스트에서 제거하고, 그 항목을 반환합니다. i가 지정되지 않으면, 리스트의 마지막 항목을 제거하고 반환합니다.
```python
fruits = ['apple', 'banana', 'cherry']
popped_fruit = fruits.pop(1)
print(popped_fruit)  # 'banana'
print(fruits)  # ['apple', 'cherry']
```

#### 6. clear(): 리스트의 모든 항목을 삭제합니다.
```python
fruits = ['apple', 'banana', 'cherry']
fruits.clear()
print(fruits)  # []
```

#### 7. index(x[, start[, end]]): 리스트에서 항목 x의 첫 번째 위치를 반환합니다. 선택적으로 start와 end를 지정하여 검색 범위를 제한할 수 있습니다.
```python
fruits = ['apple', 'banana', 'cherry', 'banana']
index_of_banana = fruits.index('banana', 2)  # Search starting from index 2
print(index_of_banana)  # 3
```

#### 8. count(x): 리스트에서 항목 x가 나타나는 횟수를 반환합니다.
```python
fruits = ['apple', 'banana', 'cherry', 'banana']
count_of_banana = fruits.count('banana')
print(count_of_banana)  # 2
```

#### 9. sort(key=None, reverse=False): 리스트의 항목들을 정렬합니다. key 매개변수는 정렬 기준을 지정하며, reverse=True는 내림차순 정렬을 의미합니다.
```python
numbers = [3, 1, 4, 1, 5, 9, 2]
numbers.sort()
print(numbers)  # [1, 1, 2, 3, 4, 5, 9]
```

#### 10. reverse(): 리스트의 항목 순서를 반대로 뒤집습니다.
```python
fruits = ['apple', 'banana', 'cherry']
fruits.reverse()
print(fruits)  # ['cherry', 'banana', 'apple']
```

#### 11. copy(): 리스트의 얕은 복사본을 새로운 리스트로 반환합니다. 
```python
fruits = ['apple', 'banana', 'cherry']
fruits_copy = fruits.copy()
print(fruits_copy)  # ['apple', 'banana', 'cherry']
```

## 4. 튜플 / 딕셔너리 / 집합 / 불

: 우리가 오늘 배우는 주제는 '컬렉션'이고, 컬렉션이란 값들을 모아서 저장할 수 있는 자료 구조를 말해요.

컬렉션에는 리스트 뿐만 아니라, 튜플 / 딕셔너리 / 집합이 있어요. 또한, 불이라는 자료형도 있습니다.

이번 시간에는 컬렉션 별 특징과 간단하게 사용하는 방법만 알아 볼 거에요. 나중에 다른 컬렉션이 필요할 때가 되면, 그 때 다시 배워 보는 것으로 해요.

---

### 1. 튜플 (Tuple)
: 튜플은 여러 개의 값을 하나로 묶는데 사용되는 자료형이에요. 리스트와 비슷하지만, 한 번 만들어진 튜플의 내용은 변경할 수 없다는 차이가 있어요. 즉, 튜플은 '읽기 전용' 리스트라고 생각하면 돼요! 📚

```python
my_tuple = (1, 2, 3)
print(my_tuple)
```

### 2. 딕셔너리 (Dictionary)
: 딕셔너리는 '키(key)'와 '값(value)'의 쌍으로 데이터를 저장하는 자료형이에요. 각각의 키는 고유해야 하며, 이 키를 통해 값을 빠르게 찾을 수 있어요. 마치 사전에서 단어를 찾듯이요! 📖
```python
my_dict = {'name': '뤼튼', 'age': 1}
print(my_dict['name'])
```

### 3. 집합 (Set)
: 집합은 중복된 값을 허용하지 않고, 순서가 없는 데이터의 모임이에요. 수학에서의 집합과 비슷하게, 교집합이나 합집합 같은 연산을 할 수 있어요. 🧮
```python
my_set = {1, 2, 3, 3}
print(my_set)  # {1, 2, 3}
```

## 5. 튜플, 딕셔너리, 집합을 깊게 배우지 않는 이유

### 🚨 조금 어려운 이야기일 수 있어요 🚨

튜플, 딕셔너리, 집합과 같은 것들을 잘 아는 것도 분명히 중요해요.

프로그래밍을 잘하고 싶다면, 각각의 컬렉션의 특징에 대해 정확하게 이해하고 있어야 해요. 어떤 상황이 주어지더라도, 적절한 방법을 선택하려면 모든 컬렉션을 사용할 준비 정도는 되어 있는 게 좋아요.

하지만, 프로그래밍을 이제 막 배우는 단계에서는 거의 대부분의 문제를 리스트로 해결할 수 있어요. 굳이 튜플이나 딕셔너리, 집합과 같은 것들을 사용하지 않아도, 대부분의 문제를 해결할 수 있다는 말이기도 해요.

생각해 보면, 수정도 할 수 없는 튜플은 왜 배우고 왜 사용하는 걸까요? 나중에 수정하고 싶어도, 이미 튜플을 사용했다면 되돌릴 수도 없는데 말이에요.

그러나, 튜플도 적당히 사용되는 곳이 있어요. 아래의 예시를 봐 볼게요.

---

## 튜플을 사용하면 좋은 경우
: 튜플은 한 번 생성되면 그 안의 값들을 변경할 수 없어요. 따라서, 프로그램 동안 변하지 않고 유지되어야 하는 값들을 저장할 때 사용하기 좋아요.

예를 들어, 한 해의 월별 일수를 저장하는 경우에 튜플을 사용할 수 있어요.
```python
months_days = (31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31)
print(months_days[1])  # 2월의 일수: 28
```

또, 튜플은 변경할 수 없기 때문에 딕셔너리의 키로 사용될 수 있어요. 예를 들어, 좌표를 키로 하고 그 좌표에 해당하는 값을 저장하고 싶을 때 튜플을 사용할 수 있어요.
```python
locations = { (37.5665, 126.9780): "서울시청", (35.1796, 129.0756): "부산시청" }
print(locations[(37.5665, 126.9780)])  # 서울시청
```

튜플은 위와 같은 상황에 사용이 돼요. 그런데 한 번 생각해 보면, 위의 상황에서 꼭 튜플을 써야 할까요? 리스트를 쓰고서는 문제를 해결할 수 없을까요? 아마 그건 아닐 거에요.

또, 프로그래밍 중간에 값을 수정해야 할 일이 생긴다면, 아마 튜플을 사용하는 코드를 모두 지우고 리스트를 사용하는 코드를 다시 작성해야 할 거에요. 시간도 많이 들고, 노력도 많이 들죠.

그래서 튜플이나 딕셔너리에 대해서는 지금은 깊이 있게 다루지 않을 거에요. 나중에 연습을 많이 하고 나면, 그 때 다시 튜플과 딕셔너리에 대해 깊이 있게 배워 보도록 할게요.

#### 4. 튜플과 리스트의 차이점에 대해 얘기해 보자.

매 수업이 끝난 후에는 오늘 배운 내용들을 다시 한 번 정리하는 시간을 가져볼 거예요. 오늘 배운 것들을 직접 설명해 보는 연습도 해보려고 해요.

문제를 푸는 것보다 설명하는 것이 더 어려울 수 있어요. 그래서 질문을 보고 네이버 블로그에 글을 써 보는 연습을 해볼 거예요.

오늘 배운 내용들은 처음 접하는 것들이라서 설명하는 것이 어려울 수 있어요. 하지만, 오늘 배운 것들을 설명하는 연습을 하다 보면 점차 능력이 향상될 거라고 생각해요.

어려워도 도전해 보는 습관을 가지는 것이 좋을 것 같아요 😊
